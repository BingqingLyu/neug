## 卡片3：一个"小bug"背后的真相

**卡片标题**
"加个 if 不就完了？"——并没有

**正文（两段对比）**

❌ 表面看起来
心跳忽略了 lightContext: true
加个判断条件修一下就行

✅ 图谱展开之后
这个"轻量定时器"穿过了 7 个子系统
heartbeat-runner.ts 一个文件导入了 34 个模块
它和完整消息处理管线走的是同一条路

**底部一句话**
每次迭代"顺手"多接一根管，债就这样悄悄积累 📈

## 卡片6：图谱 vs 传统工具

**卡片标题**
为什么这些问题平时看不见？

**正文（三条 bullet）**

🔍 grep 能找到调用点
但不知道调用者分布在几个模块

🔍 静态分析能抓语法
但看不见"这个模块被半个系统耦合着"

🔍 每次PR改动都合理
只有全局图铺开才能看见债务在哪里积累

**底部一句话**
结构性的问题，需要结构性的工具才能看见
